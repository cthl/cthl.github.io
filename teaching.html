Prerequisites

Familiarity with C, basic knowledge of Unix/Linux command line tools and Git. Key features of C will be revisited in class.

Students will required to create a private repository on GitHub for homework submissions (free for students, see here).

I highly recommend using a virtual machine (VM) for the homework, as the course requires software that students may find difficult to install, especially on MacOS and Windows (GCC, GDB, Valgrind, MPI, PETSc). A Linux-based VM with all required software is available here. It can be used with VirtualBox on all major operating systems and requires about 5 GB of free disk space.

Recommended reading and reference manuals

Slides and code examples presented in class are available here.

This course is based on the following texts, which are recommended, but not required:

  • Introduction to High Performance Computing for Scientists and Engineers by Hager and Wellein (2011)
  • The C Programming Language (2nd ed.) by Kernighan and Ritchie (1988)
Additionally, students may want to consult the following reference manuals:
  • MPI: A Message-Passing Interface Standard - Version 3.1
  • OpenMP Application Programming Interface
  • Debugging with GDB
  • Valgrind manual

Homework

Students are encouraged to discuss the homework and to work together, except on pledged problems. However, each student is expected to implement solutions and write reports individually.

Submissions must include a Makefile that can be used to compile any submitted code. Submitted code must be written in C or C++ (any standard supported by GCC). If C++ is used, please ensure that

  1. Makefiles are adjusted accordingly, e.g., use g++ -std=c++11 instead of gcc -std=c99, and
  2. the C++ code does not violate any interfaces specified in the homework problems. For example, if the task is to implement a function

    double norm2(const double *vector, int size)

    you may implement that function in C++, but you may not change its signature to something like

    double norm2(const std::vector<double> vector).
